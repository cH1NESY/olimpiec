# Решение проблемы с расходящимися ветками

## Проблема
Git не знает, как объединить ваши локальные изменения с удаленными. Нужно указать стратегию объединения.

## Решение (рекомендуется - merge)

```bash
cd ~/olimpiec

# Настроить merge как стратегию по умолчанию (опционально)
git config pull.rebase false

# Или просто указать при pull
git pull origin master --no-rebase

# Если возникнут конфликты, их нужно будет разрешить вручную
```

## Альтернатива - rebase (если хотите линейную историю)

```bash
cd ~/olimpiec

# Настроить rebase как стратегию по умолчанию
git config pull.rebase true

# Или указать при pull
git pull origin master --rebase
```

## Если возникли конфликты

После merge/rebase могут быть конфликты. Git покажет файлы с конфликтами:

```bash
# Посмотреть статус
git status

# Открыть файлы с конфликтами и разрешить их вручную
# Затем:
git add <файлы_с_разрешенными_конфликтами>
git commit
```

## Быстрое решение (если локальные изменения не важны)

Если ваши локальные изменения не критичны и вы хотите просто получить последнюю версию:

```bash
cd ~/olimpiec

# Сохранить текущее состояние (на всякий случай)
git stash

# Получить последнюю версию
git fetch origin
git reset --hard origin/master

# Если нужно вернуть изменения
git stash pop
```
